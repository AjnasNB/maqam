Research source layer

Ordered backends, explicit enforcement.

The source registry borrows useful routing and diagnostic ideas from capability-layer projects while keeping Maqam's narrower security boundary visible.

  • ResearchDocument v1

    Validated, immutable source records with adapter and channel provenance, HTTP or HTTPS URI, text or Markdown, authors, timestamps, metadata, and citations.

  • ResearchSourceRegistry

    Deterministic priorities and preferences, attempt records, authenticated-backend opt-in, fail-closed fatal errors, and a required ToolCaller for governed routing.

  • Offline source doctor

    Independent timeout-bound adapter checks with ready, degraded, unavailable, blocked, and error status.

  • No hidden process behavior

    Adapter declarations contain no command, shell, cookie, package-installation, or implicit-login fields.

Crawler and feeds

More reach inside the existing limits.

  • RSS 2.0 and Atom

    A pure offline parser bounds input, entries, metadata, and text; rejects DTD and entity declarations; sanitizes content; resolves only safe HTTP or HTTPS links; and records hashes.

  • Feed discovery

    The crawler can opt into linked RSS and Atom feeds while retaining DNS pinning, redirect re-authorization, robots handling, request, byte, origin, depth, and duration ceilings.

  • Hardened CLI

    Explicit request, depth, byte, duration, retry, feed, and origin controls plus detailed output, statistics, fail-on-error, and reliable output completion.

Still deliberately boundedMaqam does not render JavaScript, extract browser cookies, bypass access controls, rotate proxies, auto-install platform tools, or operate a distributed crawler fleet.

Agent Reach inspiration

Architecture ideas, not a coverage claim.

Agent Reach is an MIT-licensed Python capability layer with broader platform-specific installation, routing, and diagnostic coverage. Maqam 0.3.0 independently implements ordered adapters and offline checks for a TypeScript governance boundary. It does not claim equivalent channel coverage, automatic setup, cookie reuse, login support, or partnership.

Teams can compose the projects by wrapping one reviewed upstream capability in a static Maqam adapter and routing the real call through explicit policy. Direct calls remain outside Maqam.

Release verification

Close the loop on the exact artifact.

  1. Run the full test suite, clean consumer type check, website check, production audit, and package dry run.
  2. Inspect the packed file list and ensure no credentials, local state, or unrelated generated assets are included.
  3. Publish through npm Trusted Publishing only after exact maintainer approval.
  4. Verify npm version, gitHead, integrity, provenance, and public install from a clean consumer.
  5. Tag the exact source commit and attach release notes after registry verification.

This page intentionally does not invent an integrity digest or source commit before publication. The registry and tagged release are the authoritative post-publish records.
