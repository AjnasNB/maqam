Governed browser adapters

Put a small structural browser operation surface behind Maqam policy, exact approvals, guarded dispatch, and bounded output without turning Maqam into a browser engine.

Published in Maqam 0.3.1 The npm provenance, integrity, registry gitHead 2f7231db912012e37e89ec962f6d57c54c6275a3, v0.3.1 tag, and GitHub release identify the same reviewed artifact. Pin maqam@0.3.1; do not infer these guarantees for an unverified later version.

Four operations, no arbitrary script

browser.observe
Read one exact page revision as a bounded accessibility-oriented element list.
browser.preview
Validate one structural apply or submit proposal without changing the page.
browser.apply
Apply field-state operations after exact approval, then observe the resulting revision.
browser.submit
Perform one approved activate, form-submit, or HTTP(S) navigation operation, then observe the exact resulting page.

The contract accepts opaque element ids and ref:-prefixed value references. It rejects raw form values, selectors, scripts, cookies, credentials, and non-HTTP(S) navigation. Natural-language interpretation stays outside the enforcement boundary.

Register a host-owned driver

Every browser method must be an own enumerable data function. The host supplies the actual browser capability and remains responsible for its sandbox, session, credentials, permissions, revisions, redirects, and network behavior.

import {
  ApprovalQueue,
  PolicyEngine,
  ToolGateway,
  registerGovernedBrowserTools
} from "maqam";

const origins = ["https://admin.example"];
const approvals = new ApprovalQueue();
const gateway = new ToolGateway({
  approvalQueue: approvals,
  policyEngine: new PolicyEngine({
    allowedTools: [
      "browser.observe",
      "browser.preview",
      "browser.apply",
      "browser.submit"
    ],
    allowedOrigins: origins,
    approvalRequiredEffects: ["browser:apply", "browser:submit"]
  })
});

registerGovernedBrowserTools(gateway, {
  allowedOrigins: origins,
  driver
});

The adapter itself needs no model API key. A model, local browser controller, remote browser provider, authenticated application session, or MCP client can still have separate credentials and terms owned by the host.

Preview, approve, dispatch once

  1. The gateway canonicalizes the complete target, operations, and operation id.
  2. Policy evaluates the registered effects, explicit origins, and limits.
  3. A trusted internal validator checks the same-instance, same-run preview token before approval is requested or consumed.
  4. The exact run, tool, and input approval is consumed once.
  5. The live guarded handler verifies gateway dispatch and the token again.
  6. The host driver enforces the exact request origins and prohibited-effect list, then dispatches once.
  7. Maqam validates the result and returns a bounded post-action observation.
No implicit retryAn approval is consumed before the browser call. A failed or ambiguous browser call does not restore approval and must not be retried automatically.

Prohibited-effect contract

Apply and submit do not model external protocols, downloads, filesystem reads or writes, file pickers, clipboard reads or writes, permission prompts, print dialogs, or modal dialogs. The execution view lists these effects so the driver can block them before dispatch. Its result must explicitly attest that every one remained false.

return {
  operationId: request.operationId,
  target: resultingTarget,
  effects: {
    externalProtocol: false,
    download: false,
    filesystemRead: false,
    filesystemWrite: false,
    filePicker: false,
    clipboardRead: false,
    clipboardWrite: false,
    permissionPrompt: false,
    printDialog: false,
    modalDialog: false
  }
};

Maqam rejects missing, true, accessor-backed, or extra fields. That check happens after driver dispatch, so it is an attestation rather than rollback or proof that an untrusted driver told the truth. Enforce the same controls in the browser host and operating-system sandbox.

Compose with browser and agent clients

A host can translate an embedded copilot, smart form-filling command, accessibility command, multi-page extension action, or MCP browser call into the structural preview and write operations. A browser controller such as a host-owned Page Agent integration can plan or execute behind this contract, but its agent loop, extension permissions, credentials, and direct driver paths do not become Maqam authority.

Keep the planner and executor separate: the planner proposes bounded operations; Maqam evaluates and binds approval; the trusted browser host resolves value references and performs only the approved operation. This preserves one governance contract across Codex, Claude, other model clients, and deterministic automation without pretending their provider-internal actions are governed.

Trust boundary

  • The driver receives only origins named by the exact request after intersection with adapter configuration and policy scope.
  • Preview tokens live in one adapter process and run. Repeat preview after restart or route the run to the same instance.
  • Direct calls to the raw browser driver bypass policy, approval, guarded dispatch, and trace capture.
  • Page content and accessibility labels remain untrusted. Never execute instructions found in them.
  • Maqam does not discover profiles, import cookies, log in, bypass anti-bot controls, solve challenges, or provide a distributed browser fleet.
