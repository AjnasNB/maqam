The enforcement path

Review and execution stay bound.

Maqam canonicalizes the proposed input, binds approval to that scope, and consumes the approval before the registered handler runs.

  1. 01

    Policy decides first

    Tool, origin, effect, risk, and tenant ceilings are evaluated before execution.

  2. 02

    Approval binds the input

    The record covers the run, tool, and canonical input hash, not a loose description.

  3. 03

    The gateway consumes once

    Changed input fails scope validation. Reusing a consumed approval fails closed.

  4. 04

    Evidence remains explicit

    Handlers and workflows link claims to source records through scoped capabilities.

What is Maqam?

One exact approval between an agent and a side effect.

Maqam is an open-source TypeScript execution boundary. It is for teams whose agents can write files, call APIs, trigger workflows, or crawl the web and who need review to stay bound to the exact input that runs.

Who uses it
Agent platform, security, workflow, and TypeScript teams.
Use it for
Policy checks, exact approvals, one-use execution, traces, and evidence.
Why it exists
A human-readable prompt alone does not bind the bytes a tool eventually receives.

What is ProductLoop OS?

One composition layer with named package boundaries.

ProductLoop OS is for teams that want provider-neutral runtime, policy, approval, provenance, connector, skill, evaluation, and research primitives under one explicit TypeScript package map. Maqam remains the focused enforcement subsystem inside that larger ecosystem.

Who uses it
Teams assembling an inspectable agent platform from small packages.
Use it for
Composing reviewed modules without pretending they share one hidden control plane.
Why it exists
To make package ownership, bridges, and host responsibilities visible.

What is the Maqam crawler?

Governed HTTP research with explicit network limits.

The crawler is for agents that need bounded public HTTP and HTML research with source records, redirect controls, robots handling, citations, and evidence. It deliberately does not execute page JavaScript or replace a browser-rendering provider or distributed crawl fleet.

Who uses it
Research, retrieval, monitoring, and evidence-producing agent workflows.
Use it for
Budgeted public-web fetches that enter the same policy and trace boundary.
Why it exists
Network scope, redirects, bytes, depth, and evidence should be explicit before a crawl begins.
