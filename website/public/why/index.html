Category comparison

Use Maqam for the execution boundary.

Keep your orchestrator, model, provider, policy service, or crawler. Route the real side effect through a boundary that can bind approval to the exact input and consume it once.

Run public 0.2.4Inspect the boundary

The value

A missing layer, not a replacement for everything.

Maqam is strongest when an application already has useful agents or tools but needs a tighter connection between policy, review, execution, traces, and evidence.

Exact scope
Approval binds run id, registered tool, and canonical input hash.
One use
A consumed approval cannot authorize a second execution.
Detached execution
The handler receives the validated snapshot that policy reviewed.
Evidence links
Workflows can connect claims to same-run source records through scoped capabilities.
Local proof
The public 0.2.4 proof needs Node >=20.18.1 and no model key or hosted account.
Modular expansion
ProductLoop OS adds public runtime, policy, approval, provenance, skill, connector, evaluation, and browser-research packages.

When Maqam is not the answer

Choose a specialist when the primary problem is browser rendering, distributed crawling, graph scheduling, hosted reviewer operations, general-purpose policy, identity, or operating-system isolation.

Bypass remains bypassMaqam governs calls that pass through its registered gateway. Direct SDK, MCP, shell, browser, credential, or network paths need independent removal or enforcement.
