Inspect it. Break it. Propose the fix.

Maqam and ProductLoop OS are MIT licensed. Contributions arrive through reviewed pull requests, not direct writes to protected branches.

The contribution path

Keep each change narrow enough to review and strong enough to verify.

  1. 01

    Open or choose an issue

    Describe the behavior, boundary, and evidence needed before writing a large change.

  2. 02

    Fork and branch

    Create a topic branch in your fork. Contributors do not need direct write access to the upstream repository.

  3. 03

    Test the boundary

    Add a focused regression test, run the complete suite, and update security or migration notes when behavior changes.

  4. 04

    Open a pull request

    Maintainers review code, provenance, license impact, compatibility, and release claims before merge.

git clone https://github.com/YOUR_NAME/maqam.git
cd maqam
git switch -c fix/focused-boundary-test
npm ci
npm test
npm run test:consumer-types
npm audit --omit=dev
Do not include secretsRemove API keys, npm credentials, private URLs, customer data, and provider transcripts before attaching logs or fixtures.

Good first contribution shapes

Start with a bounded change that has a clear pass condition.

  • Documentation fixture

    Add a complete example for one existing API and verify every command in a clean project.

  • Adapter conformance case

    Model one failure mode in a local fixture without contacting a real provider.

  • Type declaration edge case

    Reproduce a strict TypeScript consumer problem and add it to the clean-consumer test.

  • Crawler parser case

    Add a bounded public-format fixture without bypass, login, paywall, CAPTCHA, or private-network behavior.

  • ProductLoop package example

    Improve one package example while preserving its explicit host and security boundaries.

Review standards

  • Tests cover the changed behavior and important failure paths.
  • Security-sensitive code fails closed and cannot downgrade handler effects or risk.
  • Public examples run from a clean consumer project.
  • Third-party source, licenses, notices, and modifications are recorded before incorporation.
  • Benchmarks are rerun when fingerprinted sources change.
  • Marketing claims remain narrower than the attached evidence.
